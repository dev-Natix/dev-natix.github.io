XML Service 2.0

O XML Service Henry 2.0 lê um arquivo XML por ID de comanda e toma a decisão de liberar ou bloquear a passagem na catraca Henry, operando como serviço do Windows (inicialização automática).

Como o XML Service funciona

1) Evento da catraca

A catraca envia um evento com o ID da comanda. O serviço extrai esse ID e aplica debounce para evitar repetições.

2) Leitura do XML

O serviço busca na pasta xmls/ um arquivo com o padrão <ID>.xml. Aceita tanto o formato curto (123.xml) quanto com zeros à esquerda até 20 dígitos (00000000000000000123.xml).

3) Decisão e comando

Com base no atributo EVENTO do XML (L ou C), envia o comando de liberação ou bloqueio para a catraca via TCP/IP.

Há um modo opcional por timestamp, permitindo múltiplos arquivos por ID (ID_YYYYMMDDHHMMSS.xml), onde o mais recente é utilizado.

XML padrão (gerado pelo seu sistema de caixa/PDV)

Salve como xmls/<ID>.xml. Exemplo:
<?xml version='1.0' encoding='utf-8'?>
<DATAPACKET><ROWDATA><ROW ID_COMANDA="0000000123" EVENTO="L" /></ROWDATA></DATAPACKET>
EVENTO: use L para liberar ou C para consumo/bloquear. O serviço pode excluir automaticamente o XML após liberação/bloqueio conforme configuração.

Funcionalidades

Configuração

  • Conexão TCP/IP configurável (IP, porta, tempos).
  • Pasta dos XMLs configurável (xmls/ por padrão).
  • Política quando o XML está ausente: bloquear ou liberar.
  • Pós-processamento: excluir XML após liberar e/ou bloquear.
  • Mensagens personalizáveis enviadas à catraca.
  • Direção do comando: saída, entrada ou ambos.

Operação

  • Roda como serviço do Windows (inicia automaticamente).
  • Leve e com resposta mais rápida em relação à versão anterior.
  • Reconexão automática com backoff progressivo.
  • Evita ações duplicadas.
  • Logs com rotação e detalhamento de eventos.
  • Suporte a múltiplas instâncias (pastas e nomes de serviço distintos).

Compatibilidade de nome de arquivo

  • Busca tolerante: aceita 123.xml e 00000000000000000123.xml (até 20 dígitos).
  • Leitura do XML no formato DATAPACKET > ROWDATA > ROW com atributos ID_COMANDA e EVENTO.
